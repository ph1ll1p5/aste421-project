# Running the IMUDeployment

## Prerequisites

- Raspberry Pi 4 with BNO055 IMU connected via I2C (SDA → Pin 3, SCL → Pin 5)
- I2C enabled on the Pi (`sudo raspi-config` → Interface Options → I2C)
- Python virtual environment set up on your machine
- Mac: [Rancher Desktop](https://rancherdesktop.io/) installed
- Windows: WSL (Windows Subsystem for Linux) installed

---

## Step 1: Cross-Compile for Raspberry Pi

Choose the method for your operating system.

### macOS (Docker)

macOS requires Docker to cross-compile since the ARM/Linux cross-compilers are not available natively. Install [Rancher Desktop](https://rancherdesktop.io/) and pull the F´ ARM container:

```bash
docker pull nasafprime/fprime-arm:latest
```

Start the container with your project mounted:

```bash
docker run --platform=linux/amd64 --net host -e USER=$USER -u "`id -u`:`id -g`" \
  -v "/path/to/aste421-project:/project" -it nasafprime/fprime-arm:latest
```

Inside Docker, install dependencies and build:

```bash
cd /project/asteIMU
sudo pip install cmake --upgrade
sudo pip install -r /project/lib/fprime/requirements.txt -U --force-reinstall
fprime-util generate aarch64-linux
fprime-util build aarch64-linux --all
```

> If you have already installed dependencies in a previous session and are using the same container, you can skip the `pip install` steps and go straight to `fprime-util generate`.

### Windows (WSL)

Windows users can cross-compile natively using WSL (Windows Subsystem for Linux). First, install WSL if you haven't already by running this in PowerShell:

```powershell
wsl --install
```

Then open your Ubuntu WSL terminal and install the ARM cross-compiler toolchain:

```bash
sudo mkdir -p /opt/toolchains
sudo chown $USER /opt/toolchains
curl -Ls https://developer.arm.com/-/media/Files/downloads/gnu-a/10.2-2020.11/binrel/gcc-arm-10.2-2020.11-x86_64-aarch64-none-linux-gnu.tar.xz | tar -JC /opt/toolchains --strip-components=1 -x
```

Verify the toolchain installed correctly:

```bash
/opt/toolchains/bin/aarch64-none-linux-gnu-gcc -v
```

Then build from inside WSL:

```bash
export ARM_TOOLS_PATH=/opt/toolchains
cd /path/to/aste421-project/asteIMU
fprime-util generate aarch64-linux
fprime-util build aarch64-linux --all
```

## Step 3: Run the Flight Software on the Pi

SSH into the Pi:

```bash
ssh <user>@<pi-ip>
```

Run the deployment binary:

```bash
./asteIMU_IMUDeployment -a 0.0.0.0 -p 50000
```

You should see output like:

```
[INFO] IMU open successful
Listening for single client at 0.0.0.0:50000
```

Leave this terminal open.

## Step 4: Run the GDS

In a new terminal, activate the virtual environment:

```bash
source path/to/aste421-project/fprime-venv/bin/activate
```

Navigate to the deployment directory and start the GDS:

```bash
cd path/to/aste421-project/asteIMU/IMUDeployment
fprime-gds -n \
  --dictionary ../../build-artifacts/aarch64-linux/asteIMU_IMUDeployment/dict/IMUDeploymentTopologyDictionary.json \
  --ip-client --ip-address <pi-ip> --ip-port 50000
```

The GDS will open in your browser at `http://127.0.0.1:5000`. The Pi terminal should print:

```
Accepted client at 0.0.0.0:50000
```

## Step 5: Initialize the IMU

In the GDS **Commands** tab, send the following commands in order:

1. `asteIMU.imuManager.ACCELEROMETER_RANGE_PRM_SET` → select `RANGE_2G`
2. `asteIMU.imuManager.GYROSCOPE_RANGE_PRM_SET` → select `RANGE_250DEG`
3. `asteIMU.imuManager.ACCELEROMETER_RANGE_PRM_SAVE`
4. `asteIMU.imuManager.GYROSCOPE_RANGE_PRM_SAVE`

## Step 6: View IMU Telemetry

In the GDS **Channels** tab, look for:

```
asteIMU.imuManager.Reading
```

This channel will display live accelerometer and gyroscope data from the BNO055 IMU.

## Troubleshooting

**IMU open failed** — Check that I2C is enabled on the Pi (`sudo raspi-config`) and that the correct I2C bus is set in `Main.cpp` (`/dev/i2c-1` for the standard GPIO pins).

**GDS not connecting** — Make sure the flight software binary is running on the Pi before starting the GDS. Verify the Pi's IP address with `hostname -I` on the Pi.

**No IMU readings in Channels tab** — Make sure you have sent both parameter set commands in the GDS Commands tab after connecting.

**I2C errors in Events tab** — Verify your wiring: VCC → Pin 1 (3.3V or 5V), GND → Pin 6, SDA → Pin 3, SCL → Pin 5.

**WSL toolchain not found** — Make sure `ARM_TOOLS_PATH` is set correctly and the toolchain binary exists at `/opt/toolchains/bin/aarch64-none-linux-gnu-gcc`.
